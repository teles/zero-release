# JSON Output

Use `--json` to print a stable machine-readable release result.

```bash
zero-release --dry-run --json
zero-release analyze --json
zero-release doctor --json
```

JSON output is generated by Bash helpers and escaped without `jq`.

## Release example

```json
{
  "released": true,
  "previousVersion": "1.2.3",
  "nextVersion": "1.3.0",
  "previousTag": "v1.2.3",
  "nextTag": "v1.3.0",
  "bump": "minor",
  "channel": "stable",
  "dryRun": true,
  "currentBranch": "main"
}
```

## No-release example

```json
{
  "released": false,
  "reason": "No release-worthy commits found",
  "dryRun": true,
  "currentBranch": "main"
}
```

## GitHub Action outputs

When `$GITHUB_OUTPUT` exists, zero-release also writes:

| Output | Description |
|---|---|
| `released` | Whether a release was produced |
| `version` | The next version |
| `tag` | The next Git tag |
| `bump` | `major`, `minor`, or `patch` |
| `previous-version` | The previous version |
| `previous-tag` | The previous Git tag |
| `channel` | `stable` or the prerelease channel |
