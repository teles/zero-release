# CLI Reference

The CLI is configured through flags and environment variables. No project configuration file is loaded.

## Commands

```bash
zero-release
zero-release analyze
zero-release doctor
```

The default command performs the full release flow.

`analyze` calculates the release result and exits before mutations.

`doctor` checks the environment and configuration.

## Flags

| Flag | Default | Description |
|---|---|---|
| `--dry-run` | auto in PRs | Analyze, calculate, and preview without modifying files, creating tags, pushing, committing, or calling network plugins |
| `--json` | `false` | Print stable JSON output generated by zero-release without requiring `jq` |
| `--debug` | `false` | Print debug logs to stderr |
| `--no-push` | `false` | Create local release work but do not push Git tags or branches |
| `--no-tag` | `false` | Do not create a tag |
| `--branches` | `main,master` | Comma-separated stable branches |
| `--plugins` | `release-notes` | Comma-separated plugins |
| `--tag-format` | `v%s` | Tag format with one `%s` placeholder |
| `--changelog-file` | `CHANGELOG.md` | Changelog file path |
| `--package-json` | `package.json` | Package file path |
| `--prerelease-branches` | empty | Comma-separated prerelease branches. Entries may be `branch` or `branch:channel` |
| `--prerelease-branch` | empty | Repeatable single prerelease branch mapping |

## Examples

```bash
zero-release --dry-run
zero-release --json
zero-release --branches main,master
zero-release --plugins release-notes,changelog,package-json,github-release
zero-release --tag-format "v%s"
zero-release --changelog-file NEWS.md
zero-release --package-json package.json
zero-release --branches main --prerelease-branches alpha,beta,rc
```

## Doctor

```bash
zero-release doctor
zero-release doctor --json
```

Doctor checks:

- Bash and Git availability;
- repository state;
- tag availability;
- current branch detection;
- stable and prerelease branch allowlists;
- remote/provider detection;
- plugin names;
- optional network plugin requirements and secrets;
- GitHub pull request dry-run safety.

Doctor does not create releases, tags, commits, pushes, or network calls.

## Environment context for plugins

Plugins receive release context through `ZERO_RELEASE_*` environment variables, including:

| Variable | Meaning |
|---|---|
| `ZERO_RELEASE_RELEASED` | Whether a release is being produced |
| `ZERO_RELEASE_PREVIOUS_VERSION` | Previous version |
| `ZERO_RELEASE_NEXT_VERSION` | Next version |
| `ZERO_RELEASE_PREVIOUS_TAG` | Previous Git tag |
| `ZERO_RELEASE_NEXT_TAG` | Next Git tag |
| `ZERO_RELEASE_BUMP` | `major`, `minor`, or `patch` |
| `ZERO_RELEASE_DRY_RUN` | Whether dry-run is active |
| `ZERO_RELEASE_NO_PUSH` | Whether pushes are disabled |
| `ZERO_RELEASE_NO_TAG` | Whether tag creation is disabled |
| `ZERO_RELEASE_CURRENT_BRANCH` | Current branch |
| `ZERO_RELEASE_CHANNEL` | `stable` or prerelease channel |
| `ZERO_RELEASE_NOTES_FILE` | Path to generated release notes |
| `ZERO_RELEASE_CHANGELOG_FILE` | Changelog path |
| `ZERO_RELEASE_PACKAGE_JSON` | Package JSON path |
